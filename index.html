Control Points Assistant

Control Points Assistant

Assistant block for normal control-point generation (t=0) launched directly from the overlay.

All neighbors

Runs the broader neighbor search. Turning on Missing only clears it.

Missing only

Limits CP generation to missing links. It cannot be combined with Overwrite.

Exclude sky

Skips sky-heavy regions during CP generation.

Overwrite

Allows replacing existing t0 results. If Missing only is on, the overlay switches back to All neighbors.

Assistant block for vertical control-point generation (t=1) with row and frame targeting.

What the overlay does

A structural overlay on top of the PTGui Panorama Editor

Panomind works from your saved .pts project, reconstructing frame positions and control points, then exposing structure that is hard to read in the standard PTGui interface. It complements the Panorama Editor while the project itself stays in PTGui.

What you see

The overlay surfaces link quality, control-point structure, verticals, and weak areas directly on top of the Panorama Editor, so you can review project health in context.

What it solves

It cuts review time by making structural issues easier to spot at a glance and by keeping inspection and assistant actions in the same workspace. That is especially useful when you need to review large numbers of PTGui projects quickly.

How it integrates

PTGui does not expose a public API. Panomind works from the saved .pts project and PTGui state such as Configuration.xml, then redraws the overlay from that file-based state.

Operational rule

Save in PTGui to see PTGui-side changes reflected in the overlay. If Panomind writes new control points or verticals, reload the current PTGui project in PTGui so those updated .pts changes are pulled back into the editor.
